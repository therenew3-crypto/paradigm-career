## 1. 제품 한 줄 정의

취업준비생을 위한 도구로, "나를 어떻게 어필할지 감을 못 잡아 매번 **애매(세모)**해지는 문제"를,
**미친 브레인스토밍으로 진짜 강점을 찾아 → 회사마다 최적화**하는 방식으로 해결한다.

> 공통 자소서로 '세모'가 되지 말고, 자서전 하나로 회사마다 '동그라미'가 되는 것.

---

## 2. 핵심 통찰 (제품의 해자)

- 시중 자소서 AI = "무난하게 통과될 글"에 최적화 → 모두를 평범한 **세모**로 만드는 공장.
- 이 제품 = **세모를 동그라미(또렷한 매력)로** 만든다.
- **경쟁 차별점이자 리스크가 집중된 단 하나의 기능 = ① "안 받아주고 끝까지 파고드는" 브레인스토밍.**
  PRD가 명시: 시간·노력을 여기 **몰빵**.

### 2계층 자산 구조 (설계의 심장)

| 자서전 소스 (불변) | 회사별 최적화 (가변) |
|---|---|
| 한 번 깊게. 차별화·경험 조각 | 매번 빠르게. 회사 Fit에 맞춰 꺼냄 |
| 브레인스토밍 결과 | 회사 KB와 매칭한 지원서 카드 |
| 처음 한 번만 생성 | 회사 추가 시마다 교체 |

---

## 3. MVP 핵심 흐름 (= 데모에서 보여줄 한 흐름)

```
1. 흩어진 나 쏟아내기 (10개+ 막 던지기)
2. ① 막연한 말 거부 → 끝까지 파고들어 '진짜' 캐기        ← 시간 몰빵 지점
3. ② 조각 연결 → '반복되는 나' 발견
4. ③ 장점 vs 강점 분리 → 차별화 1개 확정 (본인이 "이게 나야" 선택)
   └─ 여기까지가 '자서전 소스 (불변)'
5. ④ 그 하나로 파생: 1분자소·지원동기·차별화·클로징 + 면접 답
6. ⑤ 회사 선택 → 회사 KB 기반으로 Fit한 것만 꺼내 '동그라미 지원서'
7. 회사 추가 시 6번만 반복 (소스 불변, 최적화만 교체)
```

**데모 결정타:** 같은 자서전 하나 → 서로 다른 회사 2곳 지원서가 "각 회사에 딱 맞게 다르게" 나오는 걸 라이브로.

## 4. 기술 아키텍처

| 레이어 | 선택 | 비고 |
|---|---|---|
| 프론트 | Next.js + Tailwind | 화면 3개: ① 브레인스토밍 채팅 / ② 자서전 소스 보드 / ③ 회사별 최적화 카드 |
| 백엔드 | 서버리스 API (Next.js Route Handlers) | 대화 상태 + 파이프라인 오케스트레이션 |
| AI | Claude `claude-opus-4-8` | **모델보다 정책이 핵심** — 반아첨·정밀도 게이트·반바넘·회사 Fit을 프롬프트+룰로 박는다 |
| 데이터(시드, 수작업·서버 내장) | 은신처 단어 사전 + 핵심 4종 템플릿 + 직무/회사 진짜요구 KB | 겉역할·진짜역할·원하는 강점 |
| 저장소 | 자서전 소스(불변) ↔ 회사별 최적화(가변) **분리 저장** | MVP는 메모리/파일/SQLite로 충분 |

### Claude API 사용 지침 (확인된 레퍼런스 기준)

- **모델:** `claude-opus-4-8` (1M 컨텍스트). 기본 사용.
- **사고:** `thinking: { type: "adaptive" }` — `budget_tokens`는 4.8에서 400 에러(제거됨). 깊이는 `output_config.effort`(`low|medium|high|xhigh|max`)로 조절. 파고드는 진단 단계는 `high`.
- **샘플링 파라미터 금지:** `temperature`/`top_p`/`top_k` 전송 시 400. 행동은 프롬프트로 제어.
- **프롬프트 캐싱:** 은신처 사전·KB·정책 시스템 프롬프트는 **고정 프리픽스로 두고 `cache_control: {type:"ephemeral"}`** — 매 턴 재전송 비용 절감(읽기 ~0.1x). 회사별 변동 내용은 프리픽스 뒤에 배치.
- **구조화 출력:** 차별화 후보 3종·자서전 소스·회사별 카드 추출은 `output_config: { format: { type:"json_schema", schema:{...} } }`로 강제 (프리필/`output_format`은 사용 금지 — 4.8에서 deprecated/400).
- **스트리밍:** 채팅·긴 출력은 `messages.stream()` + `.finalMessage()`. `max_tokens`는 비스트리밍 ~16000 / 스트리밍 ~64000.
- **에러 처리:** 타입별 예외 체인(`RateLimitError` → `APIStatusError` → `APIConnectionError`). 문자열 매칭 금지.
- **환경변수:** `ANTHROPIC_API_KEY`만 설정. 클라이언트는 빈 생성자(`new Anthropic()`)로 자동 해석.

## 5. 개발 계획 (24~48h, 5 페이즈)

### Phase 0 — 셋업 & 정책 설계 (2~3h) ⭐ 가장 중요
> 코드보다 **AI 정책/데이터가 제품의 본질**. 여기를 먼저 못 박는다.

- [ ] Next.js + Tailwind + API 라우트 스캐폴딩
- [ ] `@anthropic-ai/sdk` 연동, `claude-opus-4-8` 1콜 왕복 확인
- [ ] **시드 데이터 3종 작성** (핵심 자산):
  - 은신처(막연어) 사전: "꼼꼼합니다/성실합니다/열정" 등 → 거부 트리거
  - 핵심 4종 템플릿: 1분자소·지원동기·차별화·클로징
  - 직무/회사 KB 2~3개: `{ 겉역할, 진짜역할, 원하는_강점 }`
- [ ] 데이터 스키마 확정: `AutobiographySource`(불변) ↔ `CompanyOptimization`(가변) 분리
- [ ] 정책 시스템 프롬프트 초안 + 프롬프트 캐싱 프리픽스 구조 확정

### Phase 1 — ① 미친 브레인스토밍 엔진 (8~12h) 🔥 몰빵
> 제품의 해자. 가장 많은 시간.

- [ ] 채팅 UI (입력/스트리밍/단계 표시)
- [ ] **반아첨·정밀도 게이트 프롬프트:** 막연어 감지 시 "그때 무슨 생각·기분이었어요?"로 되묻고 통과 안 시킴
- [ ] 진단 트리: 답이 구체화될 때까지 파고드는 루프
- [ ] 이탈 방지용 "거울 순간"("어, 그게 나였네") 보상 메시지
- [ ] 10개+ 소재 수집 → 저장

### Phase 2 — ②③ 연결 & 차별화 도출 (6~8h)
- [ ] 조각 연결 → "반복되는 나" 패턴 추출
- [ ] 장점 vs 강점 분리, **차별화 후보 3종 제시** (json_schema 강제) + 반바넘 룰("복붙되면 탈락")
- [ ] 사용자가 "이게 나야" 1개 선택 → **자서전 소스(불변) 확정·저장**

### Phase 3 — ④ 전 문항 파생 (4~6h)
- [ ] 확정된 차별화 1개 → 4종 템플릿 + 면접 답 자동 파생
- [ ] keystone 일관성 강제: 모든 답이 같은 차별화에서 나오게

### Phase 4 — ⑤ 회사별 최적화 + 데모 마감 (4~6h)
- [ ] 회사 선택 UI → 회사 KB와 자서전 소스 매칭
- [ ] Fit한 소재만 꺼내 "동그라미 지원서" 카드 생성, 안 맞으면 "여긴 약하다" 솔직히
- [ ] **데모 시나리오 리허설:** 회사 2곳 라이브 비교, 테스터 3명 피드백

## 6. 핵심 리스크 & 방어 (PRD 기반)

| 리스크 | 방어 |
|---|---|
| AI가 막연한 말에 "좋네요!" 하고 넘어감 → 세모 양산 | 반아첨 프롬프트 + 진단 트리 + 정밀도 게이트 |
| 바넘 효과(누구에게나 맞는 가짜 강점) | 후보 3종 + "복붙되면 탈락" + 본인 선택 |
| 억지 동그라미 → 면접서 깨짐 | 무조건 "진짜 나"에서만, 회사 최적화도 KB '진짜역할' 기반 |
| keystone↔답 따로 놀면 일관성 깨짐 | 다 같은 차별화에서 뽑게 강제 |
| 끈질긴 질문에 지쳐 이탈 | 매 단계 "거울 순간" 보상 |

## 7. 성공 기준 (데모)

- 세션 후 "내 차별화 찾았다 + 다 연결된다" 인정 비율 **≥ 40%**
- 흩어진 정보 → 차별화+핵심답변 **15분**, 회사 추가 최적화 **2분**
- 테스터 3명 중 **2명** "안 애매하고 뾰족하다"
